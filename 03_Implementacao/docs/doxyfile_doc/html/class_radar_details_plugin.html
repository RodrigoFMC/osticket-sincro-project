◆ disablePlugin()

disablePlugin ( )

Disables the plugin by executing necessary actions:

  • Cleans up all instances as the plugin is being deactivated.
  • Removes all tables related to the SINCRO project.
  • Removes the list of cabins from the osTicket database.
  • Removes patches applied to osTicket classes.
  • Reverts the logo and backdrop of the osTicket system.
  • Clears company information associated with the plugin.
Returns
bool True if the plugin was disabled successfully, false otherwise.

◆ enablePlugin()

enablePlugin ( )

Enables the plugin by executing necessary actions:

  • Creates tables and populates them with information from the SINCRO project.
  • Creates a list of cabins on the OST list and populates OST list items with cabins from the SINCRO project.
  • Applies patches to certain osTicket classes to support AJAX requests for the field filtering the list of cabins.
  • Changes the logo and backdrop of the osTicket system to the ANSR logo and backdrop.
Returns
bool True if the plugin was enabled successfully, false otherwise.

◆ isActive()

isActive ( )

Checks if the plugin is active. To prevent multiple activations, since this method overrides the parent plugin class, and to avoid it being called and performing actions more than once, a check has been added to verify if the activation state of the plugin has changed. If it has changed, it calls the handleActivationChange function to handle the activation state change of the plugin, transitioning it between active and inactive states and performing necessary actions.

Returns
bool True if the plugin is active, false otherwise.

◆ makeInstanceSpecificActions()

makeInstanceSpecificActions ( )

Performs instance-specific actions for the plugin, such as restoring plugin info from backup and adding necessary fields.

This function checks if the plugin is active and performs various operations depending on the configuration and instance state. If the "save_on_deactivate" option is enabled and it is the first run, the function restores the plugin information from backup. It also adds the necessary fields to the selected form.

Returns
bool Returns true if the actions were performed successfully, false otherwise.
